1. 查看文件的两个版本之间的差异如何帮助你了解引入的错误？
直接显示两个版本的差异，可以让你快速地了解到自己在哪些地方做了变化，也就能更快的发现错误。

2. 从长远来看，能轻松该问文伯的整个历史记录将如何提高编程者的效率？
能快速知道自己的整个编程思想的改变，能看清自己为什么做了这些改变，以及这些改变的好处。这样就能加深理解。

3. 你认为，手动选择何时创建提交（像在Git中做的那样）与自动保存版本（灵Google Docs所做的那样）各有何优缺点？
自动保存版本的好处就是永远不用担心忘记提交，缺点是不会按照逻辑更改来保存文件。手动的优缺点和自运保存刚好相反

4. 何你认为一些版本控制系统（例如 Git）允许在一次提交中保存多个文件，而其他系统（例如 Google Docs）单独对待每个文件？
屁股决定脑袋，其它系统的目标就是单独解决一个文档的版本问题，而git的目的是解决编程系统的问题，自然有多个文件相关联，所以就需要保存多个文件。

5. 如何使用命令 git log 和 git diff 来查看文件的历史记录？
进入文件所在的目录后，输入git log 即可查看历史记录，按q通出查看。复制两个版本的序列号，用git diff 序列号1 序列号2 即可查看差异。

6. 使用版本控制，如何让你更有信心进行有可能会造成破坏的更改？
因为可以用git checkout随时恢复到原来的版本

7. 现在你已设置了工作空间，你想尝试使用 Git 来做些什么？
只学了版了版本控制，我想试下能不能控制我编写的python程序，以前我的方式是保存多个文件。
