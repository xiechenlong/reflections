1. 何时你会想使用远程版本库，而不是将所有工作保存在本地？
当需要与人协作时，或者仅是想分享代码给别人，又或者仅想保存到云上，
以防本地文件意外被删后丢失所有数据

2. 为何你希望始终手动地拉取更改，而不是让 Git 自动与你的远程版本库保持同步？
当需要与人协作时，若别人也更改了远程版本库，那我需要确认是否需要直接同步到本地。
如果是自动同步，多人协作时会变得比较乱。

3. 说明Fork、克隆和分支之间的不同。何时使用某种功能而不是另一种？
Fork是将别人在Github上的版本库复制到自己的账户上，这样一来就能在此
基础上进行修改。克隆是将云资源复制到本地。分支仅仅是增加了一个版本，
并不会复制任何文件，仅会在.git文件中反映。

4. 拥有远程分支的上次已知状态的副本（存储在本地）有何好处？
这副本是合并的基础，因为是共同版本。

5. 如何在不使用 Git 或 GitHub 的情况下开展协同？哪种做法更容易？哪种做法更困难？
用云盘，每次更改新建一个文件。当然用github方便多了，但其学习成本也高，对程序员
来说是很好的选择，但要是用github来与导师同学合作写论文之类的就有些麻烦了。

6. W何时你会想在独立的分支中而不是直接在 master 中进行更改？两种方法各有何好处？
如果是与人合作，当你想要改变一些内容时，就得先在独立的分支中更改，因为你还得和
对方商量好后，将更改合并到master中。这样就可以时刻保证master是稳定的，双方都
认可的版本。
而直接在master中更改的好处就是方便了许多，不需要合并。
